# 알고리즘 문제풀이 기록 - PR / 커밋 규칙

이 저장소는 알고리즘 문제풀이를 PR 단위로 기록하는 용도입니다.
PR을 작성하거나 커밋 메시지를 작성할 때 아래 규칙을 따릅니다.

## 폴더 구조
- `baekjoon/` : 백준 문제풀이
- `programmers/` : 프로그래머스 문제풀이
- `swea/` : SWEA 문제풀이
- `yet/` : 아직 풀지 못한 문제 ([yet] 접두사)

새 문제풀이 파일은 출처에 맞는 폴더에 저장합니다.

## PR 본문에 반드시 포함할 내용
1. **사용자가 접근한 방식**: 어떤 알고리즘/자료구조로, 어떻게 풀었는지
2. **개선할 점**: 코드를 보고 더 나아질 수 있는 부분 (효율성, 가독성, 엣지 케이스 등) — 항목마다 `- [ ]` 체크리스트 형식으로 작성
3. **좋은 점**: 코드에서 잘한 부분, 칭찬할 만한 접근

## 코드 평가 시 유의사항
- SWEA 코드의 한글 패키지명(`package 문제이름;`)은 지적하지 않습니다. (의도된 폴더/패키지 구성이므로 개선점으로 다루지 말 것)

## 커밋 메시지 / PR 제목의 난이도 이모지 규칙
문제 출처와 난이도에 따라 제목 맨 앞에 아래 이모지를 붙입니다.

**SWEA**
| 난이도 | 이모지 |
|---|---|
| D2 | 2️⃣ |
| D3 | 3️⃣ |
| D4 | 4️⃣ |
| D5 | 5️⃣ |
| D6 | 6️⃣ |
| D7 | 7️⃣ |

**프로그래머스 (레벨 기준)**
| 레벨 | 이모지 |
|---|---|
| Level 1 | 1️⃣ |
| Level 2 | 2️⃣ |
| Level 3 | 3️⃣ |
| Level 4 | 4️⃣ |
| Level 5 | 5️⃣ |

**기타**
| 구분 | 이모지 |
|---|---|
| SW 모의역량 (난이도 미표기) | 🔷 |
| Pro 문제 | 🟥 |

## 제목/커밋 메시지 형식
```
{이모지}: [문제 제목]
```

예시:
```
3️⃣: [체스판 다시 칠하기]
🔷: [미세먼지 안녕!]
🟥: [점프왕 쩰리스몰]
```
